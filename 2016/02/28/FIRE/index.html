FIRE：Fast Inertial Relaxation Engine for Minimizing

By Shiyun Zhang

发表于 2016-02-28

文章目录

  2. 2. FIRE与能量最小化
  3. 3. FIRE的联系与启发

FIRE算法是Erik Bitzek等人基于MD（分子动力学，molecular dynamics）提出。MC（蒙特卡罗，monte carlo）和MD是科学计算常用的两种计算系综平均的采样方法，
MC基于概率性统计，随机模拟，局限于平衡态热力学量等静态性质的计算，不能预测体系的动力学特性；MD是一种确定性方法，基于牛顿方程，可以计算动力学性质。最近听同学说在一个做KMC（Kinetic Monte Carlo）
的组里帮忙，不知道KMC有什么黑科技。

MD硬球模拟的一般步骤

硬球软球的区别在于粒子之间的相互作用势能是否为距离的连续函数，模拟步骤大概分为初始化、趋于平衡、求结果三步，具体如下：

  1. 确定系统的状态参数：粒子数N，温度T，体积分数$\phi$等等。
  2. 确定粒子的初始位置坐标{$r{i}(0)$,i = 1,…,N}和初始速度{$v{i}(0)$,i = 1,…,N}。
  3. 构造N(N-1)/2个球对的碰撞时间列表。
  4. 对下列步骤进行循环。
  5. 从碰撞时间表中决定下一个碰撞的时间差dt和碰撞球对i和j。
  6. 计算所有粒子在dt时间后的坐标，需要的话计入周期性边界条件。
  7. 计算碰撞粒子的新速度。
  8. 更新碰撞时间列表。

在相空间看体系趋于平衡的过程，就是使轨迹从任意给定的初始点运动到最可能的区域——平衡态区域——的过程。或者简单理解为，使得体系能量最小化。当满足我们的某一个停止依据（能量或力），停止迭代，
结束时刻的位形认为是局部势能最小的位形。

FIRE与能量最小化

在FIRE提出之前，Steepest Descent(SD),Conjugate Gradient(CG),Quasi-Newton(QN),Truncated Newton(TN)等都是进行能量最小化的方法，和它们比起来，FIRE和quenching类似，是使用
欧拉积分步运行带阻尼的的分子动力学方法，需要定义时间步。

算法的优劣判据体现在：a)计算速度。b)内存的占据。c)roubustness。d)复杂程度和简便性。据说FIRE在这些上完爆了其他算法。

FIRE的作者给出了一个对其算法形象生动的描述：skier在一个高低起伏的雪山谷滑雪，想从当前位置到山谷的最低点，这有无数的路线，他依据以下原则来尽可能快的抵达：

  • 如果上坡了，就停下来。
  • 每隔一段时间dt，重新调整。如果走得正确就增加dt，不确定就减少dt，dt有最大限制。
  • 很陡，惯性很大，跟着它走，惯性说了算。

具体的算法描述如下：

MD：calculate x，F=-$\nabla$E(x), and v using any common MD integrator; check for convergence

F1: calculate P = F$\centerdot$v

F2: set v $\to$ (1-a)$\centerdot$v + a$\centerdot$F/$\vert$F$\vert$$\centerdot$$\vert$v$\vert$

F3: if P>0 and the number of steps since P was negative is larger than Nmin, increase the time step dt $\to$ min(dt$\centerdot$$f{inc}$,dtmax) and decrease a $\to$ a$\centerdot$$f{a}$

F4: if P<0, decrease time step dt $\to$ dt$\centerdot$$f{dec}$, freeze the system v $\to$ 0 and set a back to $a{start}$

F5: return to MD

FIRE的联系与启发

FIRE的关键两点是提出了速度修正和可调整的时间步长的概念。能量最小化就是系统摆脱初始条件的影响，达到平衡状态。在F2步骤中，速度的修正方式和我之前在回声状态网络ESN里所接触到的调整系统状态的
公式是一样的，这种修正方式看来是普适的，以后系统需要洗刷初始条件进行调整应该都可以用这样的形式。他对时间步长概念的建立，以及之后的调整上，方法简单粗暴而高效，值得好好学习。

我在学习FIRE的时候，一直和之前计算物理课上接触到的Creutz等人提出的微正则系统的MC方法联系在一起，也许基于MD的方法是从那里面得到的启发。他不使用随机数来确定某步MC移动是否接受，而是用以下方式实现。
将所考虑的体系附加一个外部自由度，成为demon，demon在体系中游荡，和体系交换能量来改变体系的力学变量。我觉得FIRE的处理方式和他很点相似。
